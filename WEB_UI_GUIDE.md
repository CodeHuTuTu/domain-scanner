## 📖 功能详解 Feature Details

### 1️⃣ 统计卡片 Statistics Cards

顶部显示三个实时统计数据：

- **总域名数**: 数据库中所有检查过的域名总数
- **可用域名**: 当前可注册的域名数量（绿色高亮）
- **已注册域名**: 已被注册的域名数量

💡 **提示**: 统计数据每 30 秒自动刷新

---

### 2️⃣ 扫描配置面板 Scan Configuration Panel

#### 域名长度 Domain Length
- 输入数字：1-10
- 示例：3 表示生成 3 个字符的域名
- 影响：长度越大，生成的域名数量呈指数增长

#### 域名后缀 Domain Suffix
- 输入后缀：如 .com, .net, .li
- 自动补全：输入 com 会自动变为 .com
- 支持任何有效的 TLD

#### 域名模式 Domain Pattern
- **纯字母 (D)**: 只包含 a-z (如: abc.com)
- **纯数字 (d)**: 只包含 0-9 (如: 123.com)
- **字母数字 (a)**: 包含 a-z 和 0-9 (如: a1b.com)

#### 正则过滤器 Regex Filter (可选)
- 输入正则表达式过滤域名
- 示例：
  - `^abc` - 以 abc 开头
  - `^[a-z]{2}[0-9]$` - 两个字母 + 一个数字
  - `[aeiou]` - 包含元音字母

#### 查询延迟 Query Delay
- 单位：毫秒 (ms)
- 默认：1000ms (1秒)
- 建议：500-2000ms
- 说明：每次查询之间的等待时间，防止被限流

#### 工作线程数 Worker Count
- 范围：1-50
- 默认：10
- 建议：根据网络情况调整
- 说明：并发查询的线程数量

---

### 3️⃣ 结果展示面板 Results Display Panel

#### 标签页 Tabs
- **全部**: 显示所有域名结果
- **可用**: 只显示可注册的域名
- **已注册**: 只显示已注册的域名

#### 搜索框 Search Box
- 实时搜索域名
- 支持部分匹配
- 输入延迟：500ms
- 示例：搜索 "abc" 会匹配所有包含 abc 的域名

#### 域名列表 Domain List
每个域名显示：
- 域名名称（如：abc.li）
- 状态标签：绿色"可用" 或 红色"已注册"
- 元数据：模式、长度、检查时间
- 验证签名（已注册域名显示）

---

### 4️⃣ 扫描历史 Scan History

显示最近的扫描会话：
- **Session ID**: 会话编号
- **状态标签**: 
  - 🟡 运行中 (running)
  - 🟢 已完成 (completed)
- **配置信息**: 模式、长度、后缀
- **统计数据**: 总数、可用数、已注册数
- **时间戳**: 扫描开始时间

## 🎬 使用流程 Usage Workflow

### 步骤 1: 启动服务
```bash
./start.sh
```

### 步骤 2: 访问界面
打开浏览器访问: http://localhost:8080

### 步骤 3: 配置扫描
在左侧面板填写：
- 域名长度：3
- 域名后缀：.li
- 域名模式：纯字母 (D)
- 工作线程：20
- 其他保持默认

### 步骤 4: 启动扫描
点击"开始扫描"按钮

### 步骤 5: 查看结果
- 等待扫描完成（后台运行）
- 查看顶部统计数据变化
- 在右侧面板查看域名列表
- 点击"可用"标签查看可注册域名

### 步骤 6: 搜索过滤
- 在搜索框输入关键词
- 或使用标签页过滤
- 查看匹配的域名

### 步骤 7: 查看历史
- 滚动到左侧面板底部
- 查看扫描历史记录
- 了解之前的扫描配置和结果

## 💡 使用技巧 Pro Tips

### 🎯 高效扫描
1. **从小范围开始**: 先用长度 2-3 测试
2. **合理设置线程**: 网络好可以用 20-30 个线程
3. **使用正则过滤**: 缩小搜索范围，节省时间
4. **多次小扫描**: 比一次大扫描更灵活

### 🔍 精准查找
1. **组合过滤**: 使用模式 + 正则 + 长度
2. **搜索功能**: 在结果中快速定位
3. **查看已注册**: 了解域名注册模式
4. **参考历史**: 查看之前的成功配置

### ⚡ 性能优化
1. **调整延迟**: 网络好时降低到 500ms
2. **增加线程**: 但不要超过 50
3. **使用后台扫描**: 不阻塞界面操作
4. **定期清理**: 删除旧的扫描数据

### 📊 数据管理
1. **定期导出**: 通过数据库导出重要结果
2. **分类扫描**: 针对不同后缀分别扫描
3. **记录配置**: 记下成功的扫描配置
4. **备份数据**: 定期备份 PostgreSQL 数据

## 🎯 使用场景 Use Cases

### 场景 1: 寻找短域名
```
配置：
- 长度: 2-3
- 后缀: .li, .io, .ai
- 模式: 纯字母
- 线程: 30

目标: 找到简短易记的域名
```

### 场景 2: 品牌域名搜索
```
配置：
- 长度: 4-6
- 后缀: .com
- 模式: 纯字母
- 正则: ^abc (以品牌前缀开头)
- 线程: 20

目标: 找到符合品牌的域名
```

### 场景 3: 数字域名
```
配置：
- 长度: 3-4
- 后缀: .com
- 模式: 纯数字
- 线程: 50

目标: 找到数字域名（如: 888.com）
```

### 场景 4: 混合域名
```
配置：
- 长度: 5
- 后缀: .net
- 模式: 字母数字
- 正则: ^[a-z]{3}[0-9]{2}$
- 线程: 25

目标: 3个字母 + 2个数字的组合
```

## 🎓 最佳实践 Best Practices

1. **测试先行**: 先用小参数测试配置是否正确
2. **合理延迟**: 避免过快查询导致被封禁
3. **观察响应**: 根据实际响应速度调整参数
4. **保存结果**: 及时导出或记录重要域名
5. **持续监控**: 定期扫描更新域名状态
